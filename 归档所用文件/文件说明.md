# 归档所用文件说明

本目录存放用于归档 AscendC 算子开发 session 数据的工具脚本。

## pack_codex_cc_sessions.sh

跨多 Docker 容器按项目路径过滤并归档 AI agent session 数据的批量脚本，支持 Codex CLI 和 Claude Code 双模式。

### 功能

从多个 Docker 容器的 agent 数据目录中，按项目路径筛选出属于指定项目的 session，打包为 `.tar.gz` 后 scp 到本地。

- `--tool codex`：从 `/root/.codex/sessions/` 收集 `rollout-*.jsonl`，按 `payload.cwd` 匹配项目路径。
- `--tool claude`：从 `~/.claude/` 收集完整关联数据，包括 `projects/`、`file-history/`、`tasks/`、`sessions/`、`history.jsonl`、`telemetry/`，并汇总 token 用量。

### 用法

```bash
bash 归档所用文件/pack_codex_cc_sessions.sh \
    --tool claude \
    --project /home/c00959374/AscendOpGenAgent \
    --containers cjm_cann1,cjm_cann2 \
    --ssh-host npu_server \
    --staging /home/c00959374/codex_pack_tmp \
    --output archive_$(date +%Y%m%d).tar.gz \
    [--claude-home /home/c00959374/.claude]
```

### 参数

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `--project` | ✅ | — | 项目绝对路径（用于过滤） |
| `--containers` | ✅ | — | Docker 容器名，逗号分隔 |
| `--output` | ✅ | — | 本地输出 `.tar.gz` 路径 |
| `--tool` | | `codex` | `codex` 或 `claude` |
| `--ssh-host` | | `npu_server` | SSH 跳板主机名 |
| `--staging` | | `/home/c00959374/codex_pack_tmp` | 远端临时目录 |
| `--claude-home` | | `/home/c00959374/.claude` | Claude Code 数据根目录 |

### Claude Code 模式输出结构

```
merged/
└── <container>/
    ├── projects/<encoded-project>/   # session JSONL 文件
    ├── file-history/<sessionId>/     # 文件编辑历史
    ├── tasks/<sessionId>/            # 任务数据
    ├── sessions/                     # 活跃 session 元数据
    ├── history.jsonl                 # 按 sessionId 过滤
    ├── telemetry/                    # 按 sessionId 过滤，含 token 汇总
    └── transcripts/                  # 旧版兼容（存在则全量复制）
```

stdout 中会输出 token 汇总：
```
token_summary: {"messageTokens": ..., "inputTokens": ..., "outputTokens": ..., "costUSD": ...}
```
